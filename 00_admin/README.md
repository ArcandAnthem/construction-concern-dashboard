# Construction Concern Content Operations Workspace

This workspace is used to manage Arc & Anthem's monthly content system for Construction Concern.

## Purpose
This system supports a repeatable monthly workflow that produces:

- internal research summaries
- client-facing dashboard summaries
- monthly social strategy
- social posting files
- newsletter drafts and final files
- blog drafts and final files
- image prompt files
- exported PDF deliverables

## Core Output Types

### 1. Internal Research Dashboard
Used by Arc & Anthem for deeper analysis, decision-making, citations, source notes, and references.

### 2. Client Dashboard
Used to communicate the monthly research findings, what was reviewed, strategy for the month, and what content the client will see.

### 3. Social Outputs
Includes:
- monthly social strategy
- content calendars
- final posting copy
- image prompts

### 4. Newsletter Outputs
Includes:
- draft copy
- final approved copy
- Wix-ready content structure

### 5. Blog Outputs
Includes:
- draft blog copy
- final blog copy
- SEO fields
- Wix-ready structure

## Folder Intent

- `00_admin` = operational notes and workspace documentation
- `01_config` = client settings, brand rules, and reusable preferences
- `02_prompts` = reusable prompt files for research, strategy, writing, revision, and export
- `03_research` = source logs, citations, and monthly research notes
- `04_dashboards` = internal and client-facing dashboard templates and monthly dashboards
- `05_social` = strategy, calendars, and final social captions
- `06_newsletter` = newsletter drafts and final copy
- `07_blog` = blog drafts and final copy
- `08_image_prompts` = prompts for monthly image generation
- `09_exports` = exported PDFs and other client-ready files
- `10_archive` = prior months and retired assets

## Working Rules

1. Research should be separated from final copy.
2. Internal dashboards should contain citations and a references section.
3. Client dashboards should be polished, concise, and easy to understand.
4. Final social posting files should use `Caption with hashtags` as the posting field.
5. Newsletter and blog content should share the same monthly research base, but be produced in separate channel-specific files.
6. Image prompts should be created as separate deliverables when images are not generated inside the workflow.
7. Revisions should be modular whenever possible so one output can be updated without rerunning the full system.
